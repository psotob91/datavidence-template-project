# Anti-bias

Cognitive biases that distort analysis, how they surface here, and the counter-move.

| Bias              | How it shows here                                              | Counter-move                                                      |
|-------------------|---------------------------------------------------------------|-------------------------------------------------------------------|
| Confirmation      | Reading only sources that fit the expected result             | Actively seek disconfirming evidence before concluding            |
| Anchoring         | First number / hypothesis seen dominates the estimate         | Re-derive from raw data; ignore the initial figure                |
| Availability      | Over-weighting the most recent or memorable example           | Check base rates and the full sample, not vivid cases             |
| Automation        | Trusting tool / model output without checking it              | Spot-verify outputs; treat results as claims needing an anchor    |
| Sunk-cost         | Continuing a flawed approach because effort is invested       | Decide on expected value forward; past effort is irrelevant       |
| Recency           | Latest data point treated as the trend                        | Look at the whole series; distinguish noise from signal           |
| Survivorship      | Reasoning only from cases that "made it" into the data         | Ask what was filtered out before the data reached you             |

## Rule

When uncertain, default to the **weakest** claim the evidence supports — not the
most interesting one. Understate before you overstate.
