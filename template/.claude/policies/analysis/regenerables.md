# Regenerables (artifact taxonomy)

Only **input + code** go to git; everything else regenerates. This is what keeps
the repo portable and the history clean.

## Rules

- **Classify every artifact** — input / intermediate / output / scratch:
  - **Inputs** persist, are committed, and travel: data you provide **or** inputs
    the agent authored as an expert (coding schemes, mapping tables, curated
    inputs). Never auto-delete.
  - **Intermediates & outputs** are regenerable: git-ignored, rebuilt by
    `{targets}`; never versioned, never kept as `_v` copies.
  - **Scratch** (AI interpretation logs that change with the analysis) is
    disposable: not versioned; delete once written up.
  - **Hand-injected prose** in a report (rare) is non-regenerable: the human owns
    and edits it.
- **Everything is generated by code — including Word/PDF** (Quarto → docx). Hand
  prose is the rare exception.
- **Hash, don't copy.** Provenance is a checksum in `metadata/`, not a duplicate.
- **`make reset`** (`tar_destroy()` + clean derived) rebuilds from zero on demand.
- **Index regenerables** so none escapes tracking.
