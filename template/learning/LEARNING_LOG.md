# Learning Log

Append-only journal of concrete lessons learned while working on this project.
Newest entries at the bottom. **Never edit or delete past entries** — if a
lesson is later proven wrong, append a new entry that corrects it.

Distilled, reusable conclusions graduate into `PLAYBOOK.md`; raw candidate
improvements are triaged in `REFLECTOR.md`.

## Format

One entry per line:

```
[YYYY-MM-DD][topic] lesson — what happened, what we learned, what to do next time.
```

`topic` is a short tag (e.g. `renv`, `targets`, `cowork`, `quarto`, `ci`).
